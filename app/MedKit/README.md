[![Python 3.8+](https://img.shields.io/badge/python-3.8%2B-blue.svg)](https://www.python.org/downloads/)

[![Status: Experimental](https://img.shields.io/badge/Status-Experimental-orange)]()

**MedKit** is a professional-grade framework that orchestrates Large Language Models (LLMs) to perform structured medical reasoning, entity extraction, and clinical reference tasks. It wraps raw LLM capabilities into deterministic, type-safe CLI tools designed for researchers and developers.

> **⚠️ CRITICAL DISCLAIMER**: MedKit is a **research tool** for software engineers and data scientists. It is **NOT** a medical device, a doctor, or a substitute for professional medical advice. All outputs are generated by AI and must be verified by licensed professionals.

---

## 🚀 Key Features

*   **Autonomous Medical Agent**: A ReAct-based orchestrator (`medkit-agent`) that can reason through complex clinical queries by chaining specialized tools.
*   **Structured Entity Recognition**: Extract and normalize medical entities (Drugs, Pathogens, Symptoms) into standard formats.
*   **Clinical Reference Tools**: Instant access to anatomical data, drug interactions, and ICD-11 codes.
*   **Forensic & Mental Health Modules**: Specialized workflows for SANE (Sexual Assault Nurse Examiner) interviews and mental health assessments.
*   **Model Agnostic**: Built on `LiteLLM`, allowing you to swap between local models (Ollama/Gemma, Llama 3) and cloud providers (OpenAI, Anthropic) instantly.

## 📦 Installation

1.  **Clone the repository:**
    ```bash
    git clone https://github.com/csv610/LiteLLM.git
    cd LiteLLM/app/MedKit
    ```

2.  **Install as an editable package:**
    ```bash
    pip install -e .
    ```
    *This registers the `medkit-*` CLI commands globally in your environment.*

3.  **Environment Setup:**
    Ensure you have access to an LLM. By default, MedKit expects a local Ollama instance running `gemma3`.
    ```bash
    # Example: Run a local model
    ollama run gemma3
    ```
    *To use OpenAI/Anthropic, set your API keys in your environment variables.*

## ⚡ Quick Start

The fastest way to use MedKit is through the **Orchestrator Agent**. It automatically determines which tools to use.

```bash
# Analyze a complex clinical scenario
medkit-agent "Patient is a 45yo male with Type 2 Diabetes taking Metformin, complaining of a new rash. Check for drug side effects."
```

### Configuration
Customize the model and creativity on the fly:

```bash
# Use GPT-4o with higher creativity
medkit-agent "Explain the mechanism of action for Lisinopril" --model "openai/gpt-4o" --temperature 0.5
```

## 🛠️ Tool Reference

MedKit provides a suite of specialized CLI tools for direct access to specific domains.

| Domain | Command | Description | Example Usage |
| :--- | :--- | :--- | :--- |
| **Agent** | `medkit-agent` | Autonomous reasoning & tool coordination | `medkit-agent "Flu symptoms treatment"` |
| **Medical Info** | `medkit-medical` | General medical knowledge (Anatomy, Disease) | `medkit-medical anatomy "Heart"` |
| **Drugs** | `medkit-drug` | Pharmacology & Interactions | `medkit-drug "Metformin"` |
| **Recognition** | `medkit-recognizer` | Named Entity Recognition (NER) | `medkit-recognizer drug "Advil"` |
| **Codes** | `medkit-codes` | ICD-11 Code Lookup | `medkit-codes search "Asthma"` |
| **Mental Health** | `medkit-mental` | Assessment Chatbot | `medkit-mental` (Interactive) |
| **Forensics** | `medkit-sane` | SANE Interview Protocol | `medkit-sane start` |
| **Dictionary** | `medkit-dictionary` | Terminology Builder | `medkit-dictionary build` |

### Detailed Examples

#### 1. Anatomy & Disease
Generate structured reports on body parts or conditions.
```bash
medkit-medical anatomy "Liver"
medkit-medical disease "Hypertension"
```

#### 2. Drug Analysis
Understand medications, side effects, and interactions.
```bash
# Get detailed drug monograph
medkit-drug "Atorvastatin"
```

#### 3. Entity Recognition
Extract structured data from unstructured clinical text.
```bash
medkit-recognizer disease "Patient presents with signs of acute bronchitis and mild fever."
```

## 🧩 Architecture

MedKit is organized into modular packages, each responsible for a specific domain.

```text
MedKit/
├── medkit_agent/       # Orchestrator (ReAct Loop)
├── medical/            # General Medical Knowledge (Anatomy, Surgery, etc.)
├── drug/               # Pharmacology Engine
├── recognizers/        # NLP & Entity Recognition
├── med_codes/          # ICD-11 Integration
├── mental_health/      # Psychological Assessment Models
└── sane_interview/     # Forensic Interview Protocols
```

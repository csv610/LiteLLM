# Primary Health Care Provider Assistant

A command-line tool that acts as a primary health care provider to address patient questions and concerns with a generalist medical perspective.

## Overview

The Primary Health Care Provider Assistant is designed to provide clear, accessible, and helpful medical information. Unlike a specialist, a primary care provider has broad knowledge across many medical areas and focuses on general wellness, common conditions, prevention, and guiding patients on their next steps in healthcare.

## Important Medical Disclaimers

**This tool is for informational and educational purposes only.** It is not a substitute for professional medical advice, diagnosis, or treatment. Users should:

- **Always seek the advice of a physician or other qualified health provider** with any questions regarding a medical condition.
- Never disregard professional medical advice or delay in seeking it because of something generated by this tool.
- Understand that this tool is an AI and can provide inaccurate or outdated information.
- Use the information provided as a basis for discussion with a real healthcare professional.
- In case of a medical emergency, call your local emergency services (e.g., 911) immediately.

## Features

- **Generalist Perspective**: Provides broad medical knowledge without claiming specialized expertise.
- **Accessible Language**: Uses clear, non-technical language suitable for patients.
- **Structured Guidance**: Responses include understanding the concern, symptoms, self-care, when to seek care, and next steps.
- **Structured Output**: Supports Pydantic models for machine-readable responses.
- **Markdown Support**: Generates formatted markdown for easy reading.

## Installation

### Requirements

- Python 3.8+
- LiteClient and related dependencies from the parent project

### Setup

```bash
pip install -r requirements.txt
```

## Usage

### Basic Command

Ask a health-related question:

```bash
python primary_health_care_cli.py -q "I have a slight fever and a sore throat, what should I do?"
```

### Command-Line Options

| Option | Short | Description | Default |
|--------|-------|-------------|---------|
| `--query` | `-q` | **Required.** The health concern or question you want to ask | â€” |
| `--output-dir` | `-d` | Directory for output files | `outputs` |
| `--model` | `-m` | Language model to use | `ollama/gemma3` |
| `--verbosity` | `-v` | Logging level (0-4): 0=CRITICAL, 1=ERROR, 2=WARNING, 3=INFO, 4=DEBUG | 2 |
| `--structured` | `-s` | Use structured output (Pydantic model) | False |

### Examples

Ask about a symptom:
```bash
python primary_health_care_cli.py -q "Why do I feel dizzy when I stand up quickly?"
```

Get structured information about a condition:
```bash
python primary_health_care_cli.py -q "Tell me about managing hypertension" --structured
```

Use a specific model:
```bash
python primary_health_care_cli.py -q "Is it normal to have lower back pain after exercise?" -m "gpt-4"
```

## Code Architecture

- `primary_health_care.py`: Core logic for the provider persona.
- `primary_health_care_models.py`: Data models for structured responses.
- `primary_health_care_prompts.py`: System and user prompts defining the persona.
- `primary_health_care_cli.py`: Command-line interface.

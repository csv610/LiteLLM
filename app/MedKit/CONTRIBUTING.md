# Contributing to MedKit

## Workflow

1. Create a feature branch.
2. Keep module changes scoped and documented.
3. Add or update tests where behavior changes.
4. Update relevant README and `contract.md` when needed.

## Quality Gates

- Run tests before opening a PR.
- Keep prompts, models, and CLI argument docs in sync.
- Preserve medical safety disclaimers in user-facing docs.

## Documentation Requirements

- Document new CLI flags in the module README.
- Include at least one runnable example command.

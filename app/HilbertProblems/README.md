# Hilbert's 23 Problems Reference Guide

A dynamic reference tool designed to fetch, document, and manage information regarding the 23 mathematical problems proposed by David Hilbert in 1900. This project leverages the `LiteClient` to retrieve comprehensive details using Large Language Models (LLMs), specifically optimized for `ollama/gemma3`.

## Features

- **Dynamic Retrieval**: Fetches detailed information, status, and historical context for each of Hilbert's 23 problems.
- **Persistent Caching**: Automatically saves retrieved information into model-specific Markdown files (e.g., `hilbert_problems_ollama_gemma3.md`) to minimize redundant API calls.
- **Command-Line Interface**: Provides an intuitive CLI to browse summaries or dive into the details of specific problems.
- **Structured Data**: Uses robust data models to ensure consistency in the information displayed and stored.
- **Logging & Monitoring**: Comprehensive logging configuration for tracking operations and debugging.

## Project Structure

- `hilbert_problems_cli.py`: The main entry point for the command-line interface.
- `hilbert_problems.py`: Core logic for fetching, parsing, and managing the problem guide.
- `hilbert_problems_models.py`: Data models defining the structure of a Hilbert Problem.
- `logs/`: Directory containing application and CLI logs.

## Prerequisites

- Python 3.x
- Access to an LLM provider (e.g., Ollama running `gemma3`)
- `lite` package (internal dependency for LLM communication)
- `tqdm` for progress tracking

## Installation

Ensure you have the necessary dependencies installed in your environment:

```bash
pip install tqdm
# Ensure the 'lite' package is available in your Python path
```

## Usage

The guide can be accessed via the CLI script:

### Display All Problems Summary
```bash
python hilbert_problems_cli.py
```

### Display Details for a Specific Problem
```bash
python hilbert_problems_cli.py -p 1
```

### Specify a Custom Model
```bash
python hilbert_problems_cli.py -m ollama/mistral
```

## Output

Retrievals are saved in Markdown format, allowing for easy reading and version control of the mathematical documentation generated by the models.

# NobelPrizeWinners

`NobelPrizeWinners` retrieves structured information about Nobel Prize laureates for a selected category and year.

## What It Does

- Accepts a Nobel Prize category and year.
- Generates structured winner information including biographical and discovery-related fields.
- Saves the result as `<category>_<year>.json`.

## Why It Matters

The app is useful when a user needs a typed summary of laureate information rather than an unstructured answer.

## What Distinguishes It

- Validates category and year at the CLI layer.
- Uses dedicated schemas for background, career, and scientific work.
- Produces JSON output suitable for downstream processing.

## Files

- `nobel_prize_cli.py`: CLI interface.
- `nobel_prize_info.py`: high-level fetch function.
- `nobel_prize_explorer.py`: main explorer logic.
- `nobel_prize_models.py`: schemas.
- `nobel_prize_prompts.py`: prompt builder.
- `tests/`: test files.

## Usage

```bash
python nobel_prize_cli.py --category Physics --year 2023
python nobel_prize_cli.py --category Chemistry --year 2022 --model gemini/gemini-2.5-flash
```

Model behavior:

- CLI help text: default is `$NOBEL_PRIZE_MODEL` or `gemini/gemini-2.5-flash`
- If `NOBEL_PRIZE_MODEL` is unset, `nobel_prize_info.py` falls back to `gemini/gemini-2.5-flash`

## Limitations

- The content is generated by an LLM rather than retrieved from the Nobel Foundation.
- Historical and biographical claims should be verified when precision matters.
- The app is designed for summary generation, not archival research.
